# 发布前检查清单

## 适用场景

每次你改了项目列表、图片、文案或者页面样式，准备发布前都可以按这份清单过一遍。

## 发布前 7 步

### 1. 确认配置改对了

检查 `config.yaml`：

- 项目条目是否完整；
- `frontend` / `backend` 地址是否正确；
- `thumbnail` 是否符合命名规范；
- `renew` 是否处于你预期的状态。

### 2. 同步数据和图片

执行 `pnpm sync:projects`，确认：

- `tools/config.data.ts` 已更新；
- 新图已生成；
- 不需要的旧图是否需要清理。

### 3. 本地运行检查

执行 `pnpm dev` 后重点看首页：

- 卡片数量是否正确；
- 缩略图是否都能显示；
- 按钮是否显示正确；
- 页面布局是否有明显错位；
- 联系方式悬浮入口是否正常。

### 4. 执行静态检查

建议执行：

- `pnpm check`
- `pnpm build`

如果你这次改了样式或组件，也可以额外执行：

- `pnpm lint`

### 5. 检查站点基础信息

重点确认：

- `astro.config.mjs` 中的 `site` 仍然正确；
- `Header.astro` 中的外链没有写错；
- `Footer.astro` 中备案信息没有误改。

### 6. 看看有没有残留调试内容

例如：

- 临时打印日志；
- 临时测试数据；
- 未使用图片；
- 未使用文档占位内容。

### 7. 再提交

提交前别忘了：

- git hooks 会跑格式化和部分检查；
- 文档如果改了，`README.md` 和 `docs/` 最好保持一致；
- 如果这次维护流程发生变化，顺手更新文档，给未来的自己留条活路。

## 快速回顾版

如果赶时间，只记这个顺序：

1. 改 `config.yaml`
2. 跑 `pnpm sync:projects`
3. 跑 `pnpm check`
4. 跑 `pnpm build`
5. 本地确认首页效果

这五步基本覆盖了绝大多数发布前风险。
